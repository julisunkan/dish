Build a full-featured Global Recipe Web App using Flask (Python) with the following requirements:

💡 App Name

GlobalKitchen – a lightweight, global recipe browser and grocery list manager.

🧱 Core Features (No user login required):

Global Recipe Search

Use the free TheMealDB API (https://www.themealdb.com/api.php)

Allow users to search recipes by name

Display title, image, cuisine, and category

Show full ingredients and instructions on a detail page

Grocery List

Add "🛒 Add to Grocery List" button on the recipe detail page

Store ingredients in Flask session (not database, no login)

Show a grocery list page with all collected ingredients

Add a “❌ Clear Grocery List” option

Database (SQLite)

Set up a local SQLite database using SQLAlchemy

Include models for Recipe and Ingredient (for future use)

No need to store API recipes—just define the models

Progressive Web App (PWA)

Make the app installable on mobile or desktop

Include a manifest.json and service-worker.js in static/

Register the service worker from your HTML layout

📦 Tech Stack

Backend: Python + Flask

Frontend: HTML, Jinja2 (Flask templates), Bootstrap (optional)

Database: SQLite + SQLAlchemy (for local storage)

Session: Flask session to store grocery list

External API: TheMealDB (free, no API key required)

PWA: Service worker + manifest for installability/offline support
